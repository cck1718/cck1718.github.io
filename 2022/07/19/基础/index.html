重载和重写

    重载相当于一种多义词。在同一个类中实现重载的话两个方法的方法名必须相同，参数列表必须不同(数据类型不同、参数多少
不同)。
    重写与重载不同，它基于多态，是子类继承父类的方法后对其进行修改。重写必须保证子类中方法的方法名、参数列表、返回值
与父类中的方法相同，而且重写的方法不能抛出大于父类方法的异常，同时它的访问权限修饰符不能小于父类中的方法。

==和equals

    ==比较的是两个变量所指向的值，对于两个基本数据类型来说，直接使用==进行比较没有问题，但如果是两个引用类型使用==
比较的就是各自指向的内存地址。equals默认实现也是==，通常会重写equals方法用来比较两个引用数据类型的值，比如String
类中的equals方法就重写了，首先比较两个对象的内存地址， 如果相同则返回true，如果不是则继续比较每个字符，相同的话则返
回true。

内存区(待完善)

    jvm的公共内存大致分为堆、栈、方法区，堆中存放实例对象和数组，栈中存放基本数据类型和对对象的引用，方法区中存放方
法信息(方法名、方法类型 等等)、类型信息(类、接口、注解等等的有效名)，同时方法区中还有一个常量池，存放不可变的常量。
比如User user = new User（）;这行代码，User会保留在方法区，user会保留在栈中，new User()会保留在堆内存中，当代码
运行时会通过方法区的信息获取到数据类型同时在堆内存中开辟出改对象的内存空间，最后将该对象的内存地址赋给栈中的变量。

String的内存分配

    String对象的底层实现是一个加了final修饰的char数组，所以String实际上是不可变的，而且String对象存储在常量池中。
    String s1 = "str";
    String s2 = new String("str");
    这两个方法创建对象的过程不一样，第一种创建字符串时，直接在常量池中创建数据然后将内存地址赋给了s1变量，而第二种
方式是将常量池中的内存地址赋给堆内存中String对象，然后再把String对象的地址赋给s2。

interface和abstract

    抽象类中不能有变量，抽象类中的方法可以是抽象方法或者普通方法，抽象类中的抽象方法必须被继承的子类实现，抽象类不
能用来创建对象，抽象类有构造方法且只能单继承。
    接口中可以有方法和变量，但是它的变量全是被public static final修饰的，方法全被public abstract修饰，接口没有构
造方法但是可以多实现。
    抽像类一般用于定义一种模范，而接口一般用于定义一种功能。

自动装箱和自动拆箱

    每个基本数据类型都有各自对应的包装类，这些包装类创建出来的数据都是对象，而自动装箱就是再需要将基本数据类型转换
为引用数据类型时，自动将其转换，反之则是自动拆箱，那么这么做的意义是什么呢？
    众所周知集合中时不允许加入基本数据类型的，如果往集合中加入一个基本数据类型就需要将其转换为对象，而当我们需要对
这个数据进行计算的时候又需要把他转换成基本数据类型，这时自动装箱和自动拆箱的意义就体现出来了。

Integer缓存

    Integer在加载的时候自动创建了-128~127到常量池，每次调用都是直接获取数据。
