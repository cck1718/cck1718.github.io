Spring Cloud - alibaba

spring cloud alibaba是alibaba针对springcloud微服务项目提供的一站式解决方案，相当于在原有的spring cloud工具集上的扩展，提供了一些替代组件

nacos

  • nacos是alibaba提供的组件 既是服务注册中心又是统一配置中心，将两者结合起来。nacos支持服务注册与发现，实现服务注册的负载均衡，与consul不需要另外编写服务端，同时实现了配置的统一管理，使用nacos管理配置文件不需要连接远程仓库，nacos自身包含了库并且支持持久化和自动刷新。
  • nacos默认端口为8848

nacos server

  • nacos的server不需要零星编写，进入localhost:8848/nacos即可访问，默认账号密码为nacos
  • 可以通过nacos的图形化管理界面查看服务注册和配置中心，nacos三大核心：DataId、GroupId、命名空间。
  • 命名空间，可以用于分别不同微服务项目的配置文件。
  • GroupId，可以用于分别一个微服务项目下的不同模块
  • DataId，配置文件的名称，由文件名和版本组成。

nacos server 配置文件管理

  • nacos可以对配置文件进行版本控制和文件导入导出，在修改文件后会自动刷新配置文件，只需要在调用方法上加上自动刷新注解。
  • nacos的配置文件默认存储在内嵌的数据库中，但是该数据库不支持图形查看，官方另外支持mysql来实现数据的持久化，只需要在nacos配置文件中修改配置后在创建nacos数据库并将nacos提供的sql文件导入即可。

sentinel

sentinel组件对服务进行监控，能够实现流量监控、服务熔断、热点参数监控、系统规则、用户权限等控制，同时sentinelDashboard提供图形化操作页面，不需要跟hystrix一样编写代码，可以直接在图形化页面操作规则。

  • QPS 每秒请求数
  • RT 平均响应时间

sentinelDashboard

需要下载jar包后启动，默认端口时8080，默认账户密码是sentinel。通过dashboard可以实时监控资源，但是每次新增资源必须访问资源和刷新dashboard一次。同时还能直接定义各种熔断降级规则，不需要写代码。

规则

设置某资源的规则，违反规则服务就会熔断，如果超过则该资源熔断且进入等待时间，等待时间过后即可再次访问该资源

  • 流量监控：每秒内允许访问次数，超过则服务熔断。
  • 超时监控：设置最大响应时间，超过则服务熔断。
  • 热点参数规则：根据下标(0开始)获取请求中的参数，可以根据参数或者参数特定的值的出现次数来定义熔断。
  • 在方法上加入@SentinelResource声明该资源是一个sentinel资源，同时确定该资源的别名，还可以给该资源自定义处理熔断时的方法和出路业务错误的方法。

使用

  • 启动sentinelDashboard jar包
  • 在服务中引入sentinel依赖

  • 定义资源别名
